# Node Transaction Demo Requirements

## Purpose

Build a separate one-page demo for Stock Action transactions. The page combines receiving stock in and issuing stock out while using the finalized Location Code generated by the Node Location demo.

## Scope

- Keep `node-location-demo` unchanged.
- Add a new `node-transaction-demo`.
- Add navigation between `node-location-demo` and `node-transaction-demo`.
- Use mock data derived from `All Soruce/DB_Stock_Inv.xlsx`.
- Record every new requirement and component in Markdown.

## Source Lists

- `EBoard_Admin_Master`: item master.
- `SI_Location_Master`: selectable Box-level locations.
- `SI_Item_Mapping_Location`: item/location balance.
- `SI_Item_Transaction_Barcode`: transaction history.

## Weight Rules

- User inputs total weight in grams.
- `usage_kg` is the source unit weight per piece.
- Display unit weight as grams per piece.
- Convert total input grams to kg before moving stock.
- Store both user input grams and transaction kg.
- Every DB-facing stock/weight number must use 6 decimal places.
- Calculated pcs uses `Math.round(totalWeightG / unitWeightG)`.

## Transaction Rules

- IN creates a missing item/location mapping.
- OUT requires an existing item/location mapping with enough on-hand kg.
- OUT greater than on-hand is blocked before the confirm popup.
- `f_qty` stores absolute kg.
- `f_qty_sign` is `1` for IN and `-1` for OUT.
- `f_qty_effect` stores signed kg.
- `f_before_on_hand` and `f_after_on_hand` store kg.
- All kg/g DB fields must be rounded with the same 6-decimal rule before exporting or patching.
- `f_max_capacity` is not enforced in this phase.

## Confirm Popup

After pressing IN or OUT, show a confirmation popup before posting.

The popup shows:

- Mode: IN or OUT.
- Item code, item name, and unit weight.
- Location code, Row, Shelf, Level, and Box.
- User input grams.
- Converted transaction kg.
- Calculated pcs.
- Before on-hand in kg and pcs.
- Change in kg and pcs.
- After on-hand in kg and pcs.
- Current month IN/OUT before and after this transaction.

Confirm posts the transaction. Cancel closes the popup without changing stock or history.

## Acceptance Tests

- Unit weight converts from kg to grams.
- Total grams converts to kg.
- Pcs uses nearest rounding.
- IN/OUT creates pending confirmation first.
- Cancel does not mutate stock or history.
- Confirm IN updates balance and transaction history.
- Confirm OUT updates balance and transaction history.
- OUT greater than on-hand is blocked.
- Confirm popup summary calculates before, change, after, and monthly accum correctly.
